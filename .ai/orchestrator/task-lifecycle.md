# LumenAI Task Lifecycle

## Stages
open -> in-progress -> review -> done

## Definitions

### open
Task is defined but not yet actively assigned.

### in-progress
A builder/specialist agent is actively implementing.

### review
Implementation is complete and reviewer/QA is validating.

### done
Acceptance criteria met and output documented.

## Rules
- every task lives as one markdown file
- task filename should be kebab-case
- move task file between folders as it progresses
- no task is done without validation evidence
