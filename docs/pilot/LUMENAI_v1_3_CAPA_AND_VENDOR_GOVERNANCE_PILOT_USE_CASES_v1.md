# 3. CAPA Governance Pilot Use Case

## Use Case Name
CAPA Trend Intelligence and Executive Escalation

## Customer Problem
Many healthcare organizations track CAPAs as individual records, but leadership may not have a clear view of recurrence, aging risk, risk score movement, owner workload, or which CAPAs require executive review.

## LumenAI Pilot Capability
LumenAI helps identify CAPA risk movement and escalation needs by surfacing:

- Risk score movement
- Recurrence signals
- Aging risk
- Owner workload risk
- Executive review needs
- Leadership watch items
- Recommended actions

## Pilot Questions
The pilot should answer:

1. Which CAPAs show worsening risk?
2. Which CAPAs are recurring?
3. Which CAPAs are aging without timely closure?
4. Which CAPAs require executive review?
5. Which CAPA owners have workload risk?
6. Which CAPA categories should be discussed in governance meetings?
7. Which recommendations are useful to leadership?
8. Which CAPA metrics should be exported to Power BI?

## Pilot Workflow
Recommended workflow:

1. Review current CAPA tracking process
2. Identify CAPA categories for pilot review
3. Validate LumenAI CAPA trend outputs
4. Review CAPA watchlist with customer stakeholders
5. Define executive escalation criteria
6. Identify Power BI reporting requirements
7. Capture customer feedback
8. Document pilot success criteria

## Pilot Outputs
Expected outputs:

- CAPA pilot use case validation
- CAPA escalation criteria
- CAPA trend dashboard feedback
- CAPA Power BI export mapping
- CAPA governance meeting recommendations
- CAPA next-step pilot summary

---

# 4. CAPA Escalation Criteria

LumenAI CAPA records should be considered for executive review when one or more conditions are present:

- High risk score
- Worsening risk score delta
- Recurrence count above threshold
- Aging risk above threshold
- Owner workload concern
- Patient safety or regulatory relevance
- Repeat vendor linkage
- Cross-site recurrence
- Missed target completion date
- Leadership watch designation

Suggested pilot categories:

## Executive Review

Triggers:
- High-risk CAPA
- Recurring CAPA
- Aging CAPA with unresolved risk
- CAPA linked to high-risk vendor event
- CAPA with cross-site impact

## Leadership Watch

Triggers:
- Moderate risk score
- Early recurrence
- Owner workload signal
- Trend moving in unfavorable direction
- Requires follow-up before executive escalation

## Manager Follow-Up

Triggers:
- Site-level issue
- Low-to-moderate risk
- Corrective action in progress
- Requires monitoring but not escalation

# 6. Vendor Escalation Criteria

Vendor records should be considered for executive review when one or more conditions are present:

- Worsening vendor score movement
- Repeat quality event count above threshold
- High-risk event recurrence
- CAPA-linked vendor activity
- Cross-site vendor issue
- Patient care or OR disruption risk
- Vendor response delay
- Contract performance concern
- Leadership watch designation
- Repeated failure to meet expected service or quality standards

Suggested pilot categories:

## Executive Review

Triggers:
- High-risk vendor trend
- Multiple repeat events
- CAPA-linked vendor issue
- Cross-site vendor impact
- Significant worsening score delta
- Potential contract or service-level concern

## Leadership Watch

Triggers:
- Moderate vendor trend risk
- Early repeat event signal
- Vendor response concern
- Single CAPA linkage
- Requires review in vendor governance meeting

## Manager Follow-Up

Triggers:
- Site-level vendor issue
- Low-to-moderate recurrence
- Operational follow-up needed
- Issue not yet requiring executive escalation

## Step 1 — Event Identification

Inputs may include:

- Quality events

## Step 2 — Trend Classification

LumenAI classifies signals into:

- CAPA trend signal

## Step 3 — Governance Review

Customer stakeholders review:

- CAPA trend watchlist
- Vendor trend watchlist

The governance group determines:

Capture:

- Usefulness of trend signals
- Accuracy of prioritization
- Missing fields
- Dashboard feedback
- Workflow fit
- Customer success criteria
- Next-step requirements

# 11. Pilot Data Boundary

The pilot use cases should use:

- Synthetic CAPA data
- De-identified CAPA examples
- Aggregated CAPA categories
- Synthetic vendor data
- De-identified vendor examples
- Non-PHI quality event categories
- Non-sensitive governance examples

The pilot use cases should avoid:

- PHI
- Patient identifiers
- Employee-sensitive records
- Confidential vendor contract details
- Live EHR feeds
- Live instrument tracking feeds
- Unapproved customer production data
