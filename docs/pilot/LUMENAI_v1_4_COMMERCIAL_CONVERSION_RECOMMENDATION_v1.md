# 3. Commercial Decision Pathways

The pilot may result in one of five commercial pathways.

## Pathway 1 — Proceed to Commercial Proposal

Use when:
- Executive sponsor validates value
- Workflow fit is validated
- At least one major use case is strongly validated
- Data boundary is acceptable
- Commercial next step is requested or clearly supported
- Budget/procurement pathway is identifiable

Recommended action:
Prepare commercial proposal, pricing discussion, implementation scope, and customer-specific deployment plan.

---

## Pathway 2 — Proceed to Expanded Pilot

Use when:
- Customer validates value but wants broader evaluation
- More sites, departments, or stakeholders should be included
- Power BI, CAPA, or vendor use cases need more operational testing
- Security/privacy review is manageable but incomplete
- Executive sponsor supports continued evaluation

Recommended action:
Prepare expanded pilot scope, timeline, success criteria, and pilot pricing or evaluation model.

---

## Pathway 3 — Continue Discovery

Use when:
- Customer sees potential value
- More workflow mapping is needed
- More stakeholder engagement is needed
- Data boundary or security questions require clarification
- Analytics field mapping needs refinement
- Use case value is promising but not fully validated

Recommended action:
Schedule follow-up discovery, refine use cases, update scorecards, and revisit conversion recommendation.

---

## Pathway 4 — Pause

Use when:
- Timing is not right
- Budget is not available
- Executive sponsor is not ready
- Stakeholder availability is limited
- Security/privacy or procurement review requires more time
- Customer wants to revisit later

Recommended action:
Document pause rationale, preserve evidence, define re-engagement trigger, and identify future follow-up date.

---

## Pathway 5 — Not a Fit

Use when:
- Customer does not validate the problem
- Workflow fit is weak
- Value is unclear
- Data boundary blocks meaningful evaluation
- No executive sponsor or decision pathway exists
- Commercial conversion is not realistic

Recommended action:
Close the pilot as not a fit and document lessons learned.

# 5. Commercial Conversion Decision Matrix

| Decision Area | Strong Signal | Moderate Signal | Weak Signal |
|---|---|---|---|
| Executive value | Sponsor requests next step | Sponsor sees value but needs more review | Sponsor does not validate value |
| Workflow fit | Clear operational fit | Fit requires refinement | Fit unclear |
| Data boundary | Non-PHI or approved scope works | Review still needed | Boundary blocks pilot |
| Power BI value | BI team accepts model | BI needs refinement | BI rejects or cannot use model |
| CAPA value | CAPA use case validated | More examples needed | CAPA value unclear |
| Vendor value | Vendor use case validated | More workflow mapping needed | Vendor value unclear |
| ROI/value | Clear value signals | Early value signals | No value signal |
| Commercial path | Budget/procurement path identified | Path possible but unclear | No path |

# 6. Recommendation Thresholds

## Recommend Commercial Proposal

Recommended when:

- Overall pilot success score is 4.5 or higher
- Executive value score is 4 or higher
- Commercial conversion readiness score is 4 or higher
- At least one of the following is strongly validated:
  - Power BI validation
  - CAPA governance validation
  - Vendor governance validation
- No unresolved high-risk data boundary issue exists

---

## Recommend Expanded Pilot

Recommended when:

- Overall pilot success score is 3.8 to 4.4
- Customer validates value but wants broader evaluation
- At least two use cases are partially or strongly validated
- Barriers are manageable
- Executive sponsor supports next-step evaluation

---

## Recommend Continued Discovery

Recommended when:

- Overall pilot success score is 3.0 to 3.7
- Customer sees potential value
- Workflow, analytics, security, or use case questions remain
- More evidence is needed before proposal

---

## Recommend Pause

Recommended when:

- Customer timing, budget, stakeholder availability, or approval process prevents next step
- Value is not rejected but cannot move forward now
- Re-engagement trigger can be defined

---

## Recommend Not a Fit

Recommended when:

- Overall pilot success score is below 2.0
- Customer does not validate problem or value
- No sponsor or decision pathway exists
- Pilot evidence does not support expansion

# 8. Suggested Commercial Scope Options

## Option A — Analytics-First Pilot Expansion

Best when:
- Power BI validation is strongest
- BI team accepts export structure
- Executive reporting value is validated

Suggested scope:
- Power BI export refinement
- Data dictionary completion
- Executive dashboard mapping
- Reporting cadence pilot
- Governance analytics review

---

## Option B — CAPA Governance Expansion

Best when:
- CAPA validation is strongest
- Quality/CAPA leaders validate recurrence, aging, and escalation value

Suggested scope:
- CAPA trend intelligence refinement
- CAPA escalation thresholds
- CAPA governance workflow
- CAPA reporting cadence
- CAPA executive review package

---

## Option C — Vendor Governance Expansion

Best when:
- Vendor governance validation is strongest
- Supply chain/vendor leaders validate repeat event and accountability value

Suggested scope:
- Vendor trend intelligence refinement
- Vendor escalation thresholds
- Vendor business review support
- CAPA-linked vendor activity
- Vendor executive review package

---

## Option D — Combined Governance Pilot Expansion

Best when:
- Power BI, CAPA, and vendor use cases are all validated or promising
- Executive sponsor wants integrated governance visibility

Suggested scope:
- Power BI analytics
- CAPA trend intelligence
- Vendor trend intelligence
- Combined governance review
- Executive pilot reporting
- Commercial deployment planning

---

## Option E — Discovery Extension

Best when:
- Value is promising but insufficiently validated
- More workflow/security/analytics discovery is needed

Suggested scope:
- Additional stakeholder interviews
- Workflow mapping
- Data boundary review
- Use case refinement
- Updated scorecard review

# 12. Risks to Resolve Before Commercial Conversion

Common risks:

- Unclear budget owner
- Security/privacy review incomplete
- Data boundary not finalized
- Customer workflow fit only partially validated
- BI field requirements incomplete
- CAPA thresholds need refinement
- Vendor governance owner not identified
- Executive sponsor not fully engaged
- Procurement process unknown
- Commercial scope too broad

Recommended mitigation:
Document each risk, assign an owner, define a mitigation step, and confirm whether the risk blocks conversion or can be managed during expansion.
