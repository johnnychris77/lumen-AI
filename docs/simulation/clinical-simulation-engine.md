# Clinical Simulation Engine

Codename: Project Sentinel · LumenAI Inspect v2.5

## Purpose

Before an inspection's disposition is finalized, the Clinical Simulation
Engine evaluates several possible outcomes side by side — reclean,
supervisor override, repair evaluation, remove from service — and projects
their quality, operational, and clinical-risk consequences. The goal is to
show *why* one path is recommended over the others, not to make the
decision autonomously.

LumenAI remains a decision-support tool. Every simulation output carries
`human_review_required: true` and a fixed disclaimer; no output here
establishes causation, predicts a guaranteed outcome, or replaces
supervisor sign-off.

## Inputs

The engine composes signals from existing subsystems rather than
re-deriving them:

| Input | Source |
|---|---|
| Current findings | `InspectionFinding` rows, `readiness_engine.get_primary_finding_type` |
| Supervisor Rules | `disposition_engine.recommend_disposition` (the AI's actual recommendation), `DispositionOverride` history |
| Knowledge Graph | `knowledge_graph_service.explain_inspection` — the evidence/reasoning chain for this inspection |
| Clinical Memory / Historical outcomes | `similar_case_finder_service.find_similar_cases` over `ClinicalCase` |
| Digital Twin (instrument condition) | `instrument_condition_service.instrument_condition_history` |
| Inspection Coverage | `Inspection.coverage_pct` / `coverage_quality` |

## Determinism

Projected risk/impact numbers are generated by a seeded random number
generator keyed on tenant + inspection (or instrument identity), the same
pattern used by the existing P22 Digital Quality Twin
(`digital_quality_twin_service._seed`). Re-running a simulation without new
underlying data returns the same numbers — this is a modeled heuristic
projection, not a live statistical model, and is presented as such.
